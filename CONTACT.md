## Offene Fragen

Bevor Sie eine E-Mail schreiben, prüfen Sie bitte:

1. [`METHODOLOGY.md`](METHODOLOGY.md) — wissenschaftliche Methodik des HDRI
2. [`RUNBOOK.md`](apps/hdri-factory/RUNBOOK.md) — Betriebshandbuch für Operatoren
3. [`AGENTS.md`](apps/hdri-factory/AGENTS.md) — technische Architektur der Factory-Pipeline
4. [Bestehende GitHub Issues](https://github.com/syrokomskyi/hdri/issues) — vielleicht wurde Ihre Frage bereits beantwortet

## Arten der Zusammenarbeit

| Art | Beschreibung | Erster Schritt |
|---|---|---|
| **Code-Beitrag** | Bugfix, neues Feature, neue Extraktor-Regel | [CONTRIBUTING.md](CONTRIBUTING.md) lesen, dann Pull Request öffnen |
| **Datenquelle** | Neuer Branchenkatalog, regionaler Index, IHK-Daten | Issue mit Quellenbeschreibung und Datenschutzhinweis öffnen |
| **Methodik** | Vorschlag zur Änderung von Gewichten oder Regeln | Issue mit wissenschaftlicher Begründung und Referenzen öffnen |
| **Partnerschaft** | Co-Development, gefördertes Projekt, institutionelle Zusammenarbeit | [GitHub-Discussion](https://github.com/syrokomskyi/hdri/discussions) oder E-Mail über GitHub-Profil |
| **Presse / Forschung** | Interview, Zitat, Nutzung der Daten für eine Studie | Quellenangabe: *Handwerk Digital Readiness Index (HDRI), github.com/syrokomskyi/hdri* |

## Antwortzeiten

- GitHub Issues: typischerweise innerhalb von 7 Tagen
- Pull Requests: Review innerhalb von 14 Tagen
- Partnerschaftsanfragen: erste Reaktion innerhalb von 21 Tagen

Antworten erfolgen auf Deutsch oder Englisch.
